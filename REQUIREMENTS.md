# 2048 升级版 V1.0 需求文档

## 1. 项目概述

本项目是一个基于经典网页版 2048 玩法的升级版小游戏。V1.0 使用普通传统 2048 视觉风格：4 x 4 棋盘、数字方块、分数统计、最高分保存、重新开始、移动端滑动和桌面键盘操作。

在经典玩法基础上，V1.0 增加两个道具：消除和转换。项目以纯前端方式实现，适合部署到 GitHub Pages、Netlify 或 Vercel，并支持 PWA 主屏幕图标。

V1.0 暂时不做小猫小狗方块图、动物品种名称、动物动画、动物主题背景和动物专属 2048 方块。

## 2. 项目目标

1. 做一个可以直接打开、可以游玩、可以部署的网页版 2048。
2. 复用高 star 经典 2048 的成熟架构和交互逻辑，再按需求扩展。
3. 支持无上限合成，不在 2048 停止。
4. 支持消除和转换两个道具。
5. 在 iPhone Safari 上保证滑动顺畅、布局稳定。
6. 支持添加到 iPhone 主屏幕，具备基础 PWA 能力。

## 3. 技术方案

技术栈：

1. HTML。
2. CSS。
3. JavaScript。
4. localStorage。
5. Web App Manifest。
6. Service Worker。

不使用：

1. React、Vue、Angular。
2. 后端服务。
3. 数据库。
4. 登录、广告、支付。
5. 原生 iOS App。

架构参考：

1. 参考经典开源 2048 的分层方式。
2. `GameManager` 负责游戏状态和规则。
3. `Grid` 负责棋盘数据。
4. `Tile` 负责方块数据。
5. `KeyboardInputManager` 负责键盘、触摸和按钮输入。
6. `HTMLActuator` 负责 DOM 渲染。
7. `LocalStorageManager` 负责最高分和当前局面保存。

## 5. 核心玩法规则

1. 棋盘固定为 4 x 4。
2. 初始生成 2 个数字方块。
3. 玩家通过上下左右滑动或方向键移动方块。
4. 相同数字碰撞后合并为两倍数字。
5. 每个方块在一次移动中最多合并一次。
6. 有效移动后随机生成一个新方块。
7. 新方块 90% 概率为 2，10% 概率为 4。
8. 棋盘已满且没有相邻可合并方块时，游戏结束。
9. 点击重新开始后清空当前局面并开始新局。

## 6. 无上限合成规则

1. 游戏不把 2048 作为终点。
2. 合成到 2048 后不弹出胜利中断。
3. 玩家可以继续合成 4096、8192、16384 等更高数字。
4. 内部方块值直接使用数字，例如 2、4、8、2048、4096。
5. 当数字过大影响显示时，前端可以显示为指数形式，例如 `2^18`。
6. 游戏只在无路可走时结束。

## 7. 道具系统

V1.0 道具为无限使用：

1. 消除 x 无限。
2. 转换 x 无限。

界面使用 `∞` 显示道具数量，不扣减，不需要保存道具剩余数量。

### 7.1 消除

功能：删除一个非空数字方块。

流程：

1. 点击消除按钮。
2. 进入消除模式。
3. 棋盘出现明显激活提示。
4. 点击棋盘上的一个非空方块。
5. 方块播放删除反馈。
6. 方块被移除。
7. 自动退出道具模式。

限制：

1. 空格不可选择。
2. 使用后不生成新方块。
3. 使用后不加分。
4. 游戏结束后不可使用。

### 7.2 转换

功能：修改一个已有方块的数字。

流程：

1. 点击转换按钮。
2. 进入修改模式。
3. 点击一个非空方块。
4. 弹出可选数字面板。
5. 选择目标数字。
6. 方块变为目标数字。
7. 自动退出道具模式。

平衡规则：

1. 只能改成固定候选数字：2、4、8、16、32、128。
2. 不允许改成当前方块自身数字。
3. 转换最高只能改到 128。
4. 取消选择不触发修改。

## 8. 页面结构

顶部：

1. 游戏标题 `2048`。
2. 当前分数。
3. 最高分。

说明区：

1. 简短玩法提示。
2. 重新开始按钮。

道具区：

1. 消除按钮和无限数量标记。
2. 转换按钮和无限数量标记。
3. 取消道具按钮。

棋盘区：

1. 4 x 4 网格。
2. 数字方块。
3. 游戏结束浮层。

浮层：

1. 游戏结束提示。
2. 修改数字选择面板。
3. iPhone 添加到主屏幕提示。

## 9. 交互要求

1. 手机支持触摸滑动。
2. 桌面支持方向键。
3. 滑动阈值需要避免误触。
4. 棋盘滑动时避免页面滚动。
5. 按钮点击区域不能影响棋盘滑动。
6. 道具模式必须有明显视觉提示。
7. 道具模式可以取消。
8. 游戏结束后可以重新开始。

## 10. 本地存储设计

使用 localStorage。

推荐 key：

1. `plain2048.bestScore`：最高分。
2. `plain2048.gameState`：当前局面。
3. `plain2048.installTipClosed`：是否关闭过安装提示。

当前局面包含：

1. 棋盘状态。
2. 当前分数。
3. 是否游戏结束。
4. 道具为无限使用，不需要保存剩余数量。

保存时机：

1. 每次有效移动后。
2. 每次使用道具后。
3. 重新开始时清空旧局面。
4. 最高分变化时立即保存。

## 11. PWA 要求

manifest 需要包含：

1. `name`。
2. `short_name`。
3. `start_url`。
4. `scope`。
5. `display: standalone`。
6. `orientation: portrait`。
7. `theme_color`。
8. `background_color`。
9. 图标。

Service Worker 缓存：

1. `index.html`。
2. `style.css`。
3. `manifest.webmanifest`。
4. `sw.js`。
5. `assets/icons/app-icon.svg`。
6. `js/*.js` 核心脚本。

iPhone 说明：

1. 首次打开需要联网。
2. 缓存完成后核心资源保存在 Safari/WebKit 管理的浏览器存储中。
3. 用户可在 Safari 分享菜单中选择“添加到主屏幕”。
4. 缓存受 iOS 存储策略影响，不保证永久保留。

## 12. 非功能性需求

性能：

1. iPhone Safari 滑动响应及时。
2. 使用 CSS transform 完成方块移动动画。
3. DOM 数量稳定。
4. 动画保持轻量。

兼容性：

1. iPhone Safari。
2. iOS 主屏幕 standalone 模式。
3. 桌面 Chrome、Edge、Safari。

可维护性：

1. 经典 2048 逻辑和新增道具逻辑边界清晰。
2. CSS 使用传统 2048 风格，后续可再换主题。
3. PWA 文件独立，便于部署。
